Python系列之collections-容器数据类型

发表于 2020-02-18 | 更新于: 2020-05-22 | 分类于 Python ， 常用模块 | | 本文总阅读量 次
字数统计: 3.2k 字 | 阅读时长 ≈ 14 分钟

这篇文章主要学习了collections模块中的几种常见的容器数据类型，包括：defaultdict、OrderedDict、Counter、ChainMap、namedtuple、deque。

简介

The collections module includes container data types beyond the built-in types list, dict, and tuple.

collections模块中包含了内置的list、dict、tuple数据类型之外的容器数据类型，这些数据类型可以让我们更加高效地处理数据，其中比较常用的包括如下几种：

  • defaultdict：带有默认值的字典
  • OrderedDict：有序字典
  • Counter：计数器，主要用来计数
  • ChainMap：将多个字典组成一个新的单元
  • namedtuple：生成可以使用名字来访问元素内容的tuple子类
  • deque：双端队列，可以快速的从另外一侧追加和推出对象

defaultdict-带有默认值的字典

defaultdict是带有默认值的字典，从名字可以看出其主要功能是得到的字典自带默认值，这样即使key不存在，在获取这个key值的时候也不会报错；defaultdict另一个非常重要的功能是其value值的数据结构非常多样，支持默认的各种数据结构，如list、dict、int、set等。

带默认值的用法

在使用Python默认的dict数据结构时，如果遇到键值不存在的情况还访问的话那么就会报错：KeyError，而实际使用字典的过程中肯定会遇到动态添加字典内容的情况，如果是使用默认的dict数据结构就需要增加一步判断的过程，显得麻烦，而defaultdict可以生成带有默认值的字典，不管键值存不存在访问的时候都不会报错，这就省去了判断的过程。

OrderedDict-有序字典

OrderedDict是有序的字典，注意这里的有序不是说按照ASCII排序的结果，而是按照字典的插入顺序，也就是说先插入字典的顺序在前面。

在Python 3.5及之前的版本中，默认的dict是无序的，也就是不按照插入的顺序，而Python 3.6以及以后的版本中，默认的字典也是有序的，即保留了原始的插入顺序。

namedtuple-带名称的tuple

我们经常用 tuple (元组) 来表示一个不可变对象，比如用一个 (姓名, 学号, 年龄) 的元组来表示一个学生：

这里使用 tuple 没什么问题，但可读性比较差，我们必须清楚索引代表的含义，比如索引 0 表示姓名，索引 1 表示学号。但是如果元组比较长，属性比较多，这样通过记住索引和属性的对应关系就比较麻烦，这个时候就可以使用namedtuple，其类似于R中的named vector，让tuple中的每个元素都对应一个属性，这样就可以直接使用属性来访问元素值，而不是通过索引。

deque-双端队列

deque双端队列相比于list来说最大的优点就是可以实现从队列头部快速增加(appendleft)和删除对象(popleft)，而list只提供了append和pop方法来从尾部快速插入和删除元素.

list也是可以从头部添加和取出对象：

但是值得注意的是，list的这两种用法的时间复杂度是 O(n) ，也就是说随着元素数量的增加耗时呈线性上升。而使用dequ则是 O(1) 的复杂度，所以当你的代码有这样的需求的时候， 一定要记得使用deque。
